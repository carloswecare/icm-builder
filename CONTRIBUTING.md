# Contributing to ICM Builder

Thank you for your interest in improving ICM Builder! Here are a few guidelines:

## 🐛 Reporting Issues

- Check if the issue already exists
- Describe the problem clearly
- If it's about templates, include an example of what you tried
- Suggest a fix if you have one

## 📝 Improving Documentation

- **Typos or clarity issues:** Open an issue or PR
- **New examples:** We love real-world project-tree examples from different domains
- **Clarifications to the mental model:** If something in `SKILL.md` was confusing, let us know
- **Template improvements:** Suggest improvements to templates in `references/templates.md`

## 🎯 Adding Templates

If you have a new project-tree example (e.g., for SaaS, data science, marketing), submit it as a PR:

1. Add a new section to `references/templates.md` following the existing format
2. Include the folder structure, a brief description, and what goes in each layer
3. Reference it in the main README if it's particularly useful

## 🔧 Fixing the Skill Itself

If you're improving `SKILL.md`:

1. Keep the three-layer mental model clear and centered
2. Test your explanation against the templates — does it match?
3. Update the build sequence if needed
4. Keep the "Common mistakes" section up to date

## 📤 Pull Request Process

1. Fork the repo
2. Make your changes
3. Test (read your changes back — do they make sense?)
4. Open a PR with a clear description of what you're improving and why
5. We'll review and merge or discuss

## ✨ Code of Conduct

Keep it friendly, respectful, and focused on making ICM useful for everyone.

---

**Questions?** Open an issue first before starting on a large change.
